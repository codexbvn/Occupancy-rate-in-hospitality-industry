## ūüĆź Overview

The hospitality industry lives and dies by occupancy rates. Accurate forecasting allows hotels to optimize pricing, allocate staff efficiently, reduce waste, and deliver a better guest experience. Traditional methods rely on historical booking patterns ‚ÄĒ but they miss real-time environmental signals that strongly correlate with actual room occupancy.

This project bridges that gap. Using **sensor readings** (temperature, humidity, light intensity, CO‚āā) along with the **day of the week and time of day**, a trained **Random Forest classifier** predicts in real time whether a hotel room is currently occupied or vacant ‚ÄĒ with **93.06% accuracy**.

The prediction engine is served through a **Flask web application** with a fully redesigned, luxury hotel-themed frontend.

## ūü§Ė Machine Learning Model

### Dataset

The model was trained on the [UCI Room Occupancy Detection dataset](https://archive.ics.uci.edu/dataset/357/occupancy+detection), consisting of timestamped environmental sensor readings from a real office/room.

| Split | Records | Occupancy Rate |
|---|---|---|
| Training | 8,143 | ~21% |
| Test 1 | 9,752 | ~36% |
| Test 2 | 2,665 | ~43% |

### Preprocessing Pipeline

```
Raw Input
    ‚Ēā
    ‚Ėľ
‚Ď† datetime ‚Üí day_name (e.g. "Wednesday")
    ‚Ēā
    ‚Ėľ
‚Ď° LabelEncoder ‚Üí integer (e.g. Wednesday ‚Üí 6)
   [Alphabetical: Fri=0, Mon=1, Sat=2, Sun=3, Thu=4, Tue=5, Wed=6]
    ‚Ēā
    ‚Ėľ
‚ĎĘ Extract hour from datetime (0‚Äď23)
    ‚Ēā
    ‚Ėľ
‚Ď£ Assemble feature vector:
   [Temperature, Humidity, Light, CO2, HumidityRatio, day, hour]
    ‚Ēā
    ‚Ėľ
‚Ď§ MinMaxScaler.transform() using training-data ranges
    ‚Ēā
    ‚Ėľ
‚Ď• RandomForestClassifier.predict() ‚Üí 0 (Available) or 1 (Occupied)
```

### Training Pipeline

```python
# 1. Fit scaler on training data only
scaler = MinMaxScaler()
X_train_scaled = scaler.fit_transform(X_train)

# 2. Handle class imbalance with SMOTE
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X_train_scaled, y_train)

# 3. Train Random Forest
model = RandomForestClassifier(
    n_estimators=10,
    max_features='sqrt',
    criterion='gini',
    random_state=42
)
model.fit(X_resampled, y_resampled)

# 4. Persist all artifacts
pickle.dump(model,  open('model.pkl',  'wb'))
pickle.dump(scaler, open('scaler.pkl', 'wb'))
pickle.dump(le,     open('le.pkl',     'wb'))
```

### Why Random Forest?

- **Handles non-linear boundaries** ‚ÄĒ occupancy is driven by complex interactions between light, CO‚āā, and time
- **Robust to outliers** ‚ÄĒ sensor data can have spikes; ensemble trees are naturally resistant
- **Feature importance** ‚ÄĒ provides interpretability into which sensors matter most
- **No feature scaling required** ‚ÄĒ though scaling was applied here for consistency with other candidate models (SVM) tested in the original notebook

| Metric | Value |

## ūüď° API Reference

The app exposes a single route that handles both page rendering and prediction.

### `GET /`

Returns the full homepage HTML.

```
GET http://localhost:5000/
200 OK
Content-Type: text/html
```

### `POST /`

Submits sensor readings and returns the page with a prediction result embedded.

```
POST http://localhost:5000/
Content-Type: application/x-www-form-urlencoded
```

**Request body parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `date` | `datetime-local` string | ‚úÖ | Format: `YYYY-MM-DDTHH:MM` |
| `temperature` | `float` | ‚úÖ | Room temperature in ¬įC |
| `humidity` | `float` | ‚úÖ | Relative humidity in % |
| `light` | `float` | ‚úÖ | Light intensity in lux |
| `co2` | `float` | ‚úÖ | CO‚āā concentration in ppm |
| `humidity_ratio` | `float` | ‚úÖ | Humidity ratio (kg water / kg dry air) |

**Response:**

Returns the full HTML page with the prediction injected as a Jinja2 template variable. The `status` variable will be one of `available`, `occupied`, or `error`.
